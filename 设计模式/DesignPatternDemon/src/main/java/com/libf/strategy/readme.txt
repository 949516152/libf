注意事项：如果一个系统的策略多于四个，就需要考虑使用混合模式，解决策略类膨胀的问题。

策略模式：
    策略模式将多个实现相同接口的类但是具有不同行为的类封装起来，通过传入请求的不同对象实现不同的行为。
    抽象出相同不变的部份，即接口。

    定义：策略模式将可变的部份从程序中抽像分离成算法接口，在该接口下分别封装成一系列算法实现。
        简单说就是不同的算法实现类继承或实现相同的算法接口，在各自类做出不同的算法接口实现。

策略模式的关键在于实现同一个接口，不同的类仅仅是行为差异。
面向接口编程.


策略模式简单说就是实现统一接口的类具有不同的行为；


组合：
    在类中增加一个私有域，引用另外一个已有类的实例，通过调用引用实例的方法获得新的功能。
