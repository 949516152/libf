    抽象工厂模式用于处理有多个工厂的情况下，需创建工厂生成器FactoryProducer
    抽象工厂模式：
        在抽象工厂模式中，接口是负责创建一个相关对象的工厂，不需要显示指定他们的类，
            每个生成的工程都能够按工厂模式提供对象。

    优点：当一个产品族中的多个对象被设计成一起工作时，它能保证客户端始终只使用同一个产品族中的对象。

    缺点：产品族扩展非常困难，要增加一个系列的某一产品，既要在抽象的 Creator 里加代码，又要在具体的里面加代码。

抽象工厂模式中要素：
    对于相同产品系列不同产品类型的，提供不同的产品接口，包含相同系列产品的相同属性与行为；
    对于不同的产品系列提供不同的工厂接口；

    创建工厂生成器用于创建工厂实例。


    工厂接口类用于创建工厂，对于不同系列产品通过实现产品工厂接口创建各类产品工厂实现，

    通过创建产品工厂生成器，用于获取不同产品工厂。
