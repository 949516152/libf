本包学习工厂模式：
    简单工厂模式：在工厂模式中，我们创建对象是不会对客户端暴露创建逻辑，并且都是通过同一个接口来指向新创建的对象。


    简单工厂模式，是对继承自同一个父类或接口的实现类的封装，工厂类可以根据传入参数动态创建哪一个实现类。


    缺点:由于工厂模式中工厂类集中了所有实例的创建模式，违反了高内聚低耦合的原则，将全部创建逻辑集中到了一个工厂类中；
        工厂模式创建的类都是事先能构想到的，如何需要添加新类，就需要改变工厂类，
        随着工厂类创建的实例不断增加，对系统维护和拓展不利。

注意事项：作为一个创建类模式，在任何需要生成复杂对象的地方，都可以使用工厂模式。
            但复杂对象适合使用工厂模式，而简单对象只需要使用new就可以完成创建。
            如果使用工厂模式，就需要引入一个工厂类，会增加系统的复杂性。
